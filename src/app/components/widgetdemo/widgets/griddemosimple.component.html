A simple demo for the grid showing the basic features. Almost completely controlled by a distinct view model through decorators.

This example shows simple styling of the grid using colors. One can style the various parts:

  • first: The first column
  • middle: All except first and last
  • last: The last column
  • group:
  • header: The header row
  • headerButton: The container for all header cell buttons
  • headerSortButton: The two sort buttons (up/down)
  • headerRemoveButton: The column remove button
  • footer: the footer area
  • actionCell: The cell with the action buttons

One can apply any regular styles, such as 'background-color'.

The data handling is controlled by the model itself, using decorators.

The grid in demo is using this code:

<ac-datagrid [model]="model"
             [showActions]="false"
             [columnStyle]="{{ '{' }} footer: {{ '{' }} 'background-color': 'silver' },
                              first: {{ '{' }} 'background-color': 'orange' },
                              header: {{ '{' }} color: 'white', 'background-color': 'black' },
                              headerSortButton: {{ '{' }} color: 'yellow' }
                             }" >
</ac-datagrid>

It shows how to change the styles of the particular parts of the grid.

The data are coming from the following model:

import * as Validator from 'svogv';
import {{ '{' }} PercentPipe } from './pipe/percent.pipe';

export class UserViewModelList {{ '{' }}

  @Validator.Hidden()
  id = 0;

  @Validator.Display('E-Mail', 20, 'E-Mail address')
  @Validator.Required()
  @Validator.MaxLength(100)
  @Validator.Email()
  email = '';

  @Validator.Display('Phone Number', 30, 'The user\'s phone')
  @Validator.Required()
  @Validator.MaxLength(20)
  phoneNumber = '';

  @Validator.Display('User Name', 10, 'The full name')
  @Validator.Required()
  @Validator.MaxLength(100)
  userName = '';

  @Validator.Display('Age', 40, 'From 12 to 88')
  @Validator.Range(12, 88)
  @Validator.TemplateHint('AgeTemplate')
  age = 0;

  @Validator.Display('Done', 100, 'Work progress')
  @Validator.FormatPipe(PercentPipe)
  done?: number = 0;

  @Validator.Display('Active', 200, 'User is active')
  active ? = false;

}

Not all decorators are relevant for tables, some support the integrated editors. Especially useful for tables are:

  • Display: The name of the title (header cell) and tooltip
  • FormatPipe: A pipe applied to all values before showing
  • Hidden: The property does not render in the table
