A demo showing how to manipulate the cell's content using templates. Templates work on a per type base. In the demo all cells that render a boolean value are being replaced.
Common Features of Grid Component

The appearance of cells can be modified in various ways. First, the grid provides default templates for certain data types. Second, the internal behavior can be overwritten by providing a template with the same name , such as '#boolean'. Third, the model can provide a @UiHint('template') decorator, whereas 'template' is the name of a custom template. All three variants can be mixed.

Common Features of Grid Component

The grid in demo is using this code:

<ac-datagrid [model]="model"
             [showActions]="false"
             [externals]="{{ '{' }} 'AgeTemplate': ForAgeTemplate }">
  <ng-template #boolean let-item>
    <span *ngIf="item">YES</span>
    <span *ngIf="!item">NO</span>
  </ng-template>
  <ng-template #ForAgeTemplate let-item> Age: {{ '{{' }} item }} </ng-template>
</ac-datagrid>

It shows how to change the data appearance of the particular parts of the grid using templates.

The following templates are available internally:

  • string
  • boolean
  • date
  • enum
  • number

By providing a template like this with the same name the template is being overwritten: 

<ng-template #boolean let-item>

It's possible to control the template indirectly from the model. Use @UiHint('templatename'). Define the connection using the attribute externals: 

[externals]="{{ '{' }} 'templatename': templateref }">

With this the name provided by the decorator is matched to some local template. The item is being provided by let-item.

<ng-template #templateref let-item>{{ item }} </ng-template>
