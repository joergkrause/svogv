projects/svogv/src/lib/decorators/hint-formatpipe.decorator.ts

FormatPipe
FormatPipe(pipe, pipeParams)

The FormatPipe decorator. Provide the name of a Pipe that's being used by the dynamic pipe formatter. Hence, the form does not need to apply forms manually. The reason is that you may create forms automatically and hence can't write actual Pipes somewhere. This applies especially if you create a table and loop through properties.

Parameters :
Name Optional
pipe No
pipeParams No

projects/svogv/src/lib/decorators/hint-sortable.decorator.ts

Sortable
Sortable(canSort: boolean, sortCallback?)

The Sortable decorator.

The does not sort columns for properties tagged with@Sortable(false). The default is that all columsn are sortable. Either avoid this decorator or use @Sortable(true). Additionally, if the decorator is provided, you can add a sort function callback as second parameter.

Parameters :
Name Type Optional Description
canSort boolean No

Suppress or allow sorting.
sortCallback Yes

An optional callback that provides a sort instruction. If omitted, Array.prototype.sort is being used.

projects/svogv/src/lib/decorators/hint-templatehint.decorator.ts

TemplateHint
TemplateHint(template, params?)

The Template decorator.

One can define the way a property gets rendered. Currently supported:

  • TextArea
  • Calendar
  • Range
  • Number
  • Text

The Calendar creates Date-fielsd. However, in casde of a datatype Date the date field will be created anyway.

Parameters :
Name Optional Description
template No

The Name that appears in form fields as a watermark.
params Yes

Depending of template some additional values as a dictionary.
